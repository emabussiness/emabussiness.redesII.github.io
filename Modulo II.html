Módulo 2: Direccionamiento IP

Duración: 5 sesiones

Sesión 4: Direcciones IPv4 y Clases

1. Estructura de IPv4

  • 32 bits (4 octetos en formato decimal): 192.168.1.1
  • Notación binaria: 11000000.10101000.00000001.00000001

2. Clases de Direcciones

Clase Rango Máscara Default Uso
A 1.0.0.1 - 126.255.255.254 255.0.0.0 (/8) Grandes redes (Ej: ISP)
B 128.0.0.1 - 191.255.255.254 255.255.0.0 (/16) Universidades/empresas
C 192.0.0.1 - 223.255.255.254 255.255.255.0 (/24) Pequeñas redes (LAN)
D 224.0.0.0 - 239.255.255.255 - Multicast
E 240.0.0.0 - 255.255.255.255 - Experimental

Ejemplo Práctico:

  • 10.50.20.1 → Clase A (privada)
  • 172.16.254.1 → Clase B (privada)
  • 200.100.50.25 → Clase C (pública)

3. Direcciones Privadas vs. Públicas

Rango Privado Equivalente Público (NAT)
10.0.0.0 - 10.255.255.255 Cualquier IP pública
172.16.0.0 - 172.31.255.255
192.168.0.0 - 192.168.255.255

Sesión 5: Máscaras de Subred y CIDR

1. Concepto de Máscara

Ejemplo: 255.255.255.0 = /24 (24 bits para red, 8 para hosts).

2. Subneteo Básico

Ejemplo: Dividir 192.168.1.0/24 en 4 subredes:

Subred Dirección de Red Rango de Hosts Broadcast
1 192.168.1.0 192.168.1.1 - 192.168.1.62 192.168.1.63
2 192.168.1.64 192.168.1.65 - 192.168.1.126 192.168.1.127
3 192.168.1.128 192.168.1.129 - 192.168.1.190 192.168.1.191
4 192.168.1.192 192.168.1.193 - 192.168.1.254 192.168.1.255

Sesión 6: Conversión Binario-Decimal

1. Método de Conversión

Ejemplo: Convertir 192 a binario:

128 64 32 16 8 4 2 1 → Valores posicionales
1 1 0 0 0 0 0 0 → 192 = 128 + 64

2. Tabla de Referencia

Decimal Binario
192 11000000
168 10101000
1 00000001
